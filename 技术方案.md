# Shoplive 技术方案（提交版）

## 1. 方案目标与问题定义

Shoplive 面向电商商家在短视频生产中的三类核心痛点：  
1) 从商品信息到可用视频的生产链路长、人工编辑成本高；  
2) 跨平台与跨地区投放时，脚本与画面一致性难控制；  
3) 生成结果缺少可编辑闭环，难以快速二次优化并上线。

本方案通过“商品理解 -> 脚本与提示词生成 -> Veo 视频生成 -> 在线后处理导出”的一体化链路，将视频生产标准化为可重复、可扩展的流程，支持快速交付营销视频。

## 2. 总体技术方案

系统采用 **Flask 单体后端 + 原生 JS 多页面前端** 的轻量架构：后端统一托管前端静态资源，并提供 AI 能力编排 API；前端围绕 Landing、Agent、Studio、Image Lab 四个工作台组织业务流程。

- **前端层（`frontend`）**：负责交互、参数采集、任务提交、轮询与结果展示。  
- **应用层（`backend/api/*`）**：按能力域拆分路由（agent / shoplive / veo / media / video-edit）。  
- **业务编排层（`briefing.py` + `common/helpers.py`）**：输入标准化、脚本模板、提示词生成、质量校验、第三方返回解析。  
- **基础设施层（`infra.py`）**：鉴权、代理发现与容错、统一外部请求参数处理。  
- **外部 AI 服务层**：Google Vertex Veo、Google Image、LiteLLM（文本生成与改写）。

## 4. 核心业务流程

1. **输入采集与预处理**  
   用户可上传商品图、粘贴商品链接或直接输入草稿；前端对图片做基础质量检测，后端对字段进行标准化（卖点去重、时长规范化、参数补齐）。

2. **商品洞察与脚本生成**  
   `agent_api` 提供商品解析与图文洞察；`shoplive_api` 按 `validate -> generate_script -> pre_export_check/build_export_prompt` 工作流生成可执行分镜与最终提示词。

3. **视频生成与状态轮询**  
   前端调用 `/api/veo/start` 提交 Veo 任务，按间隔轮询 `/api/veo/status`，提取可播放链接并展示结果。

4. **在线编辑与导出闭环**  
   生成后进入视频编辑面板，调用 `/api/video/edit/export` 对速度、调色、文字蒙版、BGM 混音进行后处理，输出可访问成片链接（`/video-edits/<filename>`）。

## 5. 关键技术设计

- **模块化 API 设计**：按能力边界拆分路由，降低耦合，便于单点扩展。  
- **双路径兜底策略**：脚本与提示词优先走 LLM 生成，异常时回退模板生成，保证链路可用性。  
- **质量门禁机制**：对输入图片分辨率、清晰度、主体占比做规则校验，减少低质量生成。  
- **代理与鉴权容错**：支持显式代理、环境代理与本地代理探测，提升外部模型调用稳定性。  
- **可编辑结果交付**：ffmpeg 本地导出能力保障“可预览+可复用”的业务闭环。

## 6. 非功能与上线说明

- **可用性**：前后端一体部署，默认本地可启动；关键步骤有错误提示与回退逻辑。  
- **性能**：视频生成采用异步任务+轮询，前端不阻塞；编辑导出按需触发。  
- **安全合规**：服务账号凭据通过环境变量/文件注入；提示词侧包含合规约束，避免虚假与违规表达。  
- **运维**：统一后端入口，静态资源同服务托管；支持端口与代理配置，便于演示环境快速搭建。

## 7. 方案价值（面向评审）

- 将“商品素材 -> 营销视频”从多工具串联变为单平台闭环，显著降低制作门槛。  
- 通过标准化工作流与门禁规则，提高视频一致性与可交付质量。  
- 在生成后提供可编辑能力，缩短从创意到上线的迭代周期，支撑业务转化效率提升。
